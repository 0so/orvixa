# Dataset — provenance and scope (FROZEN)

**The `candles` table currently contains SYNTHETIC data only.**

## What's in the database

12 symbols (`BTC`, `ETH`, `SOL`, `BNB`, `XRP`, `AVAX`, `LINK`, `DOGE`, `PEPE`,
`SHIB`, `WIF`, `TRX`), each with 30 days of 1-minute OHLCV candles
(43,200 rows/symbol, 518,400 rows total), generated by
`scripts/generate_demo_candles.py`.

Every one of these symbols is tagged `synthetic_data` in `symbols.tags`.

## What this data IS

- A deterministic (seeded), gap-free, schema-correct candle series.
- Useful for exercising the backfill loader, the replay/analytics pipeline,
  and the M5 signal-validation harness end to end ("pipeline correctness").

## What this data IS NOT

- **Not real market data.** Prices are a seeded random walk with mild mean
  reversion — there is no real price action, news, liquidity, or market
  microstructure behind it.
- **Not market-valid for trading evaluation.** Any "edge", hit rate, or
  profitability number computed against this dataset is meaningless and
  must not be used to draw conclusions about real signal edge.

## Guardrail

`run_signal_validation` (in `src/orvixa/backtest/signal_validation.py`) calls
`classify_dataset` (in `src/orvixa/backtest/dataset.py`) before every run:

- It prints `dataset: REAL` or `dataset: SYNTHETIC` plus the requested mode.
- On a SYNTHETIC dataset, `mode="pipeline_correctness"` (the default)
  proceeds but prints an explicit warning that results do not represent
  real signal edge.
- On a SYNTHETIC dataset, `mode="edge_evaluation"` raises `ValueError` and
  refuses to run.

A symbol is classified SYNTHETIC if `"synthetic_data"` is present in its
`symbols.tags` array.

## Data layer freeze

The following are now frozen — do not modify:

- `src/orvixa/backfill/csv_loader.py`
- `src/orvixa/runners/backfill.py`
- `scripts/generate_demo_candles.py`
- the `candles` schema (alembic migrations)

This is a fixed test dataset layer. Adding new data sources, loaders, or
generators is out of scope; loading real market data requires a new,
explicitly-approved data source.
